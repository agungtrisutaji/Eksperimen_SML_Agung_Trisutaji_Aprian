Experiment summary:

1. Load the Telco Customer Churn dataset from the required raw CSV path.
2. Inspect rows, columns, duplicate records, missing values, target distribution, numeric columns, and categorical columns.
3. Convert `TotalCharges` to numeric because the raw CSV stores some values as text and blank strings.
4. Drop `customerID` because it is an identifier and not a useful predictive feature.
5. Map target `Churn`: `No -> 0`, `Yes -> 1`.
6. Split train and test data with stratification to keep target distribution stable.
7. Fit preprocessing only on training data to avoid data leakage.
8. Use median imputation and standard scaling for numeric features.
9. Use most-frequent imputation and one-hot encoding for categorical features.
10. Train baseline `LogisticRegression` and `RandomForestClassifier` models.
11. Track accuracy, precision, recall, f1_score, and roc_auc with MLflow.
12. Run lightweight hyperparameter tuning for Random Forest.
13. Select the best model by `f1_score`.
14. Serve the selected model through FastAPI.
15. Monitor serving behavior with Prometheus metrics and Grafana alerting.
