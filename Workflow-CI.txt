# Workflow CI

Repository GitHub public:

```text
https://github.com/agungtrisutaji/Membangun-Sistem-Machine-Learning
```

Repository ini digunakan untuk seluruh kriteria submission:

- eksperimen dataset;
- preprocessing workflow;
- model training;
- MLflow Project;
- CI workflow;
- Docker image;
- monitoring dan logging.

Workflow utama:

```text
.github/workflows/mlops-ci.yml
```

Workflow preprocessing:

```text
.github/workflows/preprocessing.yml
```

Trigger:

- `push`
- `pull_request`
- `workflow_dispatch`

Runner:

```text
ubuntu-latest
Python 3.11
```

Workflow membuat dan memakai `.venv` di runner:

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r Membangun_model/requirements.txt
python -m pip install -r "Monitoring dan Logging/requirements.txt"
```

MLflow Project folder:

```text
Workflow-CI/MLProject/
```

MLflow Project commands:

```bash
cd Workflow-CI/MLProject
mlflow run . -e main --env-manager local
mlflow run . -e tuning --env-manager local
```

CI steps:

1. Check required secrets without printing their values.
2. Check raw dataset presence.
3. Run `python preprocessing/automate_Agung_Trisutaji_Aprian.py`.
4. Run `python Membangun_model/telco_customer_churn_preprocessing/preprocess.py`.
5. Run MLflow Project entry point `main`.
6. Run MLflow Project entry point `tuning`.
7. Upload preprocessing and model artifacts.
8. Try Advanced Docker build with `mlflow models build-docker`.
9. Fall back to `Monitoring dan Logging/Dockerfile` if MLflow model URI is unavailable or `build-docker` fails.
10. Push Docker image only if Docker Hub secrets exist.

Advanced Docker target:

```bash
mlflow models build-docker -m "runs:/<RUN_ID>/model" -n agungtrisutaji/telco-churn-mlops
```

Fallback Docker target:

```bash
docker build -f "Monitoring dan Logging/Dockerfile" -t agungtrisutaji/telco-churn-mlops:latest .
```

Docker Hub image:

```text
agungtrisutaji/telco-churn-mlops:latest
```

Required GitHub Actions secrets:

```text
DAGSHUB_USERNAME
DAGSHUB_TOKEN
MLFLOW_TRACKING_URI
DOCKERHUB_USERNAME
DOCKERHUB_TOKEN
```

Secret values must not be printed in logs. The repository must be public for Dicoding review.
